Features

Everything you need for secure email

Enterprise-grade S/MIME capabilities integrated directly into the Outlook ribbon.

Encrypt & Decrypt

Encrypted messages display inline in the reading pane — no .p7m attachments to open. Native Outlook S/MIME compatible with Entrust and Thunderbird.

Digital Signatures

Sign outgoing emails with clear or opaque signatures. Supports sign-then-encrypt per RFC 5751 for maximum interoperability.

Certificate Manager

Browse your Windows certificate store and manage certificates for external contacts via the Certificate Contacts dialog.

LDAP Lookup

Optional LDAP directory lookup for recipient certificates. Queries are sanitized with RFC 4515 injection prevention built in.

Certificate Exchange

Share your public certificate with contacts so they can send you encrypted email. One click in the ribbon.

Security Dashboard

Animated task pane with FIPS mode indicator, configurable log levels, and GitHub issue integration for one-click bug reporting.

How It Works

Up and running in minutes

No admin privileges needed. Parcl installs per-user and integrates directly into Outlook's ribbon.

1

Install

Run the per-user installer. No administrator required — everything stays under your user profile.

2

Configure

Open the Parcl tab in Outlook and configure your LDAP directories and certificate preferences.

3

Select Certs

Choose your signing and encryption certificates from the Windows certificate store, or import new ones.

4

Send Secure

Click Encrypt or Sign in the Parcl ribbon. Recipient certificates are looked up automatically via LDAP.

Architecture

Clean, modular design

Three focused projects: the Outlook add-in, a standalone core library, and a lightweight installer.

  parcl/ 
src/
  Parcl.Addin/         # Outlook VSTO COM Add-in
    Ribbon, Dialogs, TaskPane, Animations
  Parcl.Core/          # Core library (zero Outlook dependency)
    SmimeHandler, CertificateStore, LdapCertLookup
  Parcl.Installer/     # WiX per-user installer (HKCU, no admin)

tests/
  Parcl.Core.Tests/    # xUnit tests for the core library

Security

Built with defense in depth

Parcl protects your data at every layer — from credentials to configuration to the messages themselves.

HMAC Settings Integrity

Configuration files are protected with HMAC verification to detect unauthorized tampering.

DPAPI Credential Protection

LDAP credentials are encrypted at rest using Windows Data Protection API, scoped to the current user.

PII Sanitization

Diagnostic logs automatically strip email addresses, certificate subjects, and other personally identifiable information.

FIPS 140-2 Compatible

All cryptographic operations use Windows CNG providers. FIPS mode can be enabled system-wide without code changes.

Apache 2.0 License

Fully open source with patent grant. Inspect every line, audit the cryptography, contribute improvements.

RFC 4515 Injection Prevention

All LDAP filter inputs are sanitized to prevent injection attacks against directory services.
